# /reentry — 프로젝트 복귀 맵

오랜만에 다시 연 프로젝트를 빠르게 파악합니다. 현재 세션 상태 요약이 아니라 프로젝트 자체를 다시 이해하는 용도입니다.

## 실행

1. README / ETHOS.md / AGENTS.md를 먼저 읽어 목적과 원칙을 복원합니다.
2. `git status -sb` / `git log --oneline -5` / `python3 scripts/omc.py state status --target .` 로 현재 상태를 보강합니다.
3. 아래 7개만 짧게 출력합니다.

```text
프로젝트 한 줄 요약:
핵심 구조:
실행/검증 진입점:
주의할 SSOT/금지 경로:
최근 작업 흔적:
다음 읽을 파일 3개:
추천 다음 스킬:
```

## 규칙

- 문서 기준과 현재 상태 기준을 분리합니다.
- 다음 읽을 파일은 정확히 3개만 제시합니다.
- 추천 다음 스킬은 정확히 1개만 제시합니다.
- tree dump / README 재요약 / 구현 태스크 분해는 금지합니다.

- 자동으로 진행하지는 않습니다.

프로젝트 복귀 요청: $ARGUMENTS
