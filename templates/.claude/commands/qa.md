# /qa — 수동 QA 체크리스트

구현 후 변경 범위 기준으로 수동 QA 항목을 뽑습니다. 코드 리뷰나 배포 게이트를 대신하지 않습니다.

## 실행

1. `git diff --stat HEAD`와 현재 plan/diff를 기준으로 변경 범위를 읽습니다.
2. 아래만 짧게 출력합니다.

```text
변경 기능:
영향 화면/흐름:
사용자 유형:
확인 환경:
정상 흐름:
예외 흐름:
회귀 포인트:
우선순위 높은 QA 5개:
실행 결과 아님:
추천 다음 스킬:
```

## 규칙

- 구현 후 수동 QA 체크리스트만 생성합니다.
- `/plan`의 테스트 설계, `/review`의 코드 리스크 분석, `/ship`의 배포 게이트를 대신하지 않습니다.
- diff 또는 plan 정보가 없고 입력 부족이면 `고정 QA 생성 불가`로 멈춥니다.

## 다음 추천

- 주추천 1개만 제시: 구현 후 검수면 `/review`, 배포 직전 최종 점검이면 `/ship`, 구현 전이거나 입력 부족이면 `/plan`
- 자동으로 진행하지는 않습니다.

변경 요청: $ARGUMENTS
