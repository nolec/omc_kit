# OMC Brainstorm

요구사항이 모호할 때만 씁니다. 명확하면 `$omc-plan`, 방향 검증이면 `$omc-office-hours`로 갑니다.

## Phase 0. 컨텍스트

```bash
python3 scripts/omc.py state status --target .
git branch --show-current 2>/dev/null
```

Phase 4 전 코드 작성 금지. 질문은 순서대로 하나씩 묻고, 사용자가 답한 후 다음 질문으로 넘어갑니다.

## Phase 1. What

- 지금 어떤 일이 벌어지고 있는가?
- 기대와 실제의 차이는 무엇인가?
- 왜 지금 해결해야 하는가?

산출물: 현상 한 문장.

## Phase 2. Why

- 근본 원인은 무엇이라고 보는가?
- 이전 시도는 무엇이고 왜 실패했는가?
- 이 문제를 가장 잘 아는 사람은 어떻게 설명하는가?

산출물: 원인 가설 1개 이상.

## Phase 3. How

- 가장 단순한 해결 방법은?
- 이상적인 해결책은?
- 빠른 해법과 이상적 해법의 트레이드오프는?

산출물: 해결 옵션 2개 이상.

## Phase 4. Decide

Phase 1~3 산출물이 모두 있을 때만 수렴합니다.

```text
핵심 문제:
옵션 A [빠른]:
옵션 B [균형]:
옵션 C [이상적]:
권장 옵션:
권장 이유:
사용자 확인:
다음 액션:
```

명확한 선택지가 1개뿐이면 옵션 A만 둡니다. 구현 안 함이 맞으면 권장 옵션에 `구현 안 함`을 쓰고 보류 이유를 남깁니다.

## 이후 액션

- 제품 검증 필요 → `$omc-office-hours`
- 구현 설계 가능 → `$omc-plan`
- 여전히 모호함 → Phase 1~3 재실행
- 자동 진입 금지. 사용자가 다음 스킬을 명시할 때까지 멈춥니다.
