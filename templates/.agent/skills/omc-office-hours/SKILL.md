# OMC Office Hours

코딩 전 제품 게이트입니다. 방향이 모호하면 `$omc-brainstorm`, 범위·우선순위가 흔들리면 `$omc-ceo-review`로 돌립니다.

## Phase 0. 컨텍스트 확인

```bash
python3 scripts/omc.py state status --target .
git branch --show-current 2>/dev/null
```

## 6개 질문

6개 답변 완료 전 코드 작성 금지. 빈칸 금지, 모르면 `N/A — 이유`로 씁니다.

```text
대상 기능:

Q1. 이 기능을 쓸 사람이 정확히 누구인가?
    "모든 사용자"는 답이 아닙니다.

Q2. 지금 그 사람이 겪는 실제 고통은 무엇인가?

Q3. 이 기능이 완성됐을 때 측정 가능한 성공 기준은?
    "더 편하다"는 답이 아닙니다. 근거 있는 수치나 관찰 지표를 둡니다.

Q4. MVP를 한 문장으로 쓰면?
    사용자가 완료 시한 또는 위험 평가를 판단해야 합니다.

Q5. 이 기능에서 명시적으로 제외하는 것은?

Q6. 10점짜리 버전은 무엇이며 MVP와 어떻게 다른가?
```

## 판정

- PROCEED: Q1~Q6이 모두 구체적이고, Q3이 측정 가능하며, 사용자가 시한/위험을 판단했습니다.
- RETHINK: Q1 사용자, Q2 고통, Q3 성공 기준 중 하나가 흐립니다.
- HOLD: Q3이 측정 불가이거나 Q5 제외 범위가 너무 넓습니다. `$omc-ceo-review`로 넘깁니다.

AI가 임의로 PROCEED를 결정하지 않습니다. 사용자가 판단한 시한 또는 위험 평가가 있어야 합니다.

## 이후 액션

- PROCEED면 `$omc-plan`으로 설계합니다.
- RETHINK면 흐린 질문만 다시 답합니다.
- HOLD면 `$omc-ceo-review`로 범위를 재검토합니다.
- 출력 후 자동 진입 금지. 사용자가 다음 스킬을 명시할 때까지 멈춥니다.
