# OMC 요구사항 소크라테스식 정제

AI는 **소크라테스 방식**으로 진행합니다.
**Phase 4 컨펌 전에 코드를 작성하지 않습니다.**

> **이 스킬을 쓰면 안 되는 상황**:
> 요구사항이 이미 명확하다면 바로 `$omc-plan` 을 사용하세요.
> 이 스킬은 "뭘 만들지 모르겠을 때"만 사용합니다.

---

## Phase 1 — What: 현상 파악

AI가 아래 질문을 **순서대로 하나씩** 묻는다. 사용자가 답한 후 다음 질문으로 넘어간다.

```
P1-Q1. 지금 어떤 일이 벌어지고 있는가?
        (현재 상태를 가능한 구체적으로 설명)
        → _______________

P1-Q2. 기대하는 것과 실제 상황의 차이는 무엇인가?
        → _______________

P1-Q3. 이 문제를 지금 해결해야 하는 이유는?
        (해결 안 하면 어떻게 되는가?)
        → _______________
```

☐ Phase 1 완료 — 현상이 구체적으로 정의됨

---

## Phase 2 — Why: 원인 탐색

```
P2-Q1. 이 문제가 발생하는 근본 원인은 무엇이라고 생각하는가?
        → _______________

P2-Q2. 지금까지 이 문제를 해결하려 했던 시도가 있었는가?
        있었다면 왜 실패했는가?
        → _______________

P2-Q3. 이 문제를 가장 잘 이해하는 사람은 누구이며,
        그 사람은 원인을 어떻게 설명하는가?
        → _______________
```

☐ Phase 2 완료 — 원인 가설이 1개 이상 도출됨

---

## Phase 3 — How: 해결 방향 탐색

```
P3-Q1. 이 문제를 해결하는 가장 단순한 방법은?
        (기술적으로 완벽하지 않아도 됨)
        → _______________

P3-Q2. 이상적인 해결책은 어떤 모습인가?
        (제약이 없다면?)
        → _______________

P3-Q3. 위 두 가지 방법의 트레이드오프는?
        (빠르지만 부족한 것 vs 이상적이지만 오래 걸리는 것)
        → _______________
```

☐ Phase 3 완료 — 해결 방향 2개 이상 도출됨

---

## Phase 4 — Decide: 수렴

> Phase 1~3 체크박스가 모두 체크된 후에만 이 단계로 진입합니다.

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
BRAINSTORM 결과
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
핵심 문제: _______________________________________________

구현 옵션:
  A) [빠른]    _____________________ (예상 시간: ___)
  B) [균형]    _____________________ (예상 시간: ___)
  C) [이상적]  ___________________ (예상 시간: ___)

권장 옵션: A / B / C
권장 이유: _______________________________________________

> ※ 탐색 결과 "구현하지 않는다"가 결론이면:
>    권장 옵션 칸에 "구현 안 함"을 기재하고 이유를 적은 후 Phase 4를 완료합니다.
>    → 다음 액션: `$omc-office-hours` 로 전달하거나 해당 요구사항을 보류 처리합니다.

☐ 사용자 확인 완료
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 이후 액션

| 상황 | 다음 스킬 |
|---|---|
| 방향은 잡혔지만 제품 관점 검증이 필요 | `$omc-office-hours` |
| 방향 확정, 구현 설계로 바로 진입 | `$omc-plan` |
| 아직도 모호함 — Phase 1~3 재실행 필요 | 브레인스토밍 재시작 |

## 실패 조건

- Phase 3까지 완료했는데도 "핵심 문제"를 한 문장으로 정의할 수 없으면 → 재시작
- 사용자가 Phase 4 결과를 확인하지 않으면 → `$omc-plan` 진입 금지
