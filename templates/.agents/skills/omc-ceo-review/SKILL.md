# OMC CEO Review

기능 범위와 우선순위를 재검토합니다. 사용자에게 보여줄 것: 모드 / 공통 5개 / 선택 모드 질문 2개 / 사용자 판단 / 결론 / 이유
시스템이 암묵적으로 처리: HOLD 자동 선택 금지, 근거 부족 시 중단, 다음 스킬 자동 진입 금지. 요구사항이 불명확하면 `$omc-brainstorm`, 이미 구현한 뒤면 `$omc-retro`로 갑니다.

## Phase 0. 컨텍스트

```bash
python3 scripts/omc.py state status --target .
git log --oneline -5 2>/dev/null
```

## 모드

모드와 결론은 다릅니다. 모드 미지정 시 자동 HOLD 금지, 아래 모드를 보여주고 사용자 선택을 받습니다.

- EXPAND: 범위가 너무 좁아 임팩트가 작을 때 / SELECTIVE: 일부만 골라 구현할 때
- HOLD: 지금 할지 확신이 없을 때 / REDUCE: 범위를 줄여 빠르게 검증할 때

## 공통 5개

예/아니오 한 줄 답변 금지. 각 답변은 수치 또는 근거를 포함합니다.

1. 이 기능이 없으면 사용자가 이탈하는가? 2. 핵심 가치 제안을 강화하는가? 3. 단순 버전으로도 80% 가치를 낼 수 있는가?
4. 성공 지표 1개는 무엇인가? 5. 시한 안에 가능한가? 필수 기술부채면 시한 대신 위험 평가를 쓴다.

## 선택 모드 질문 2개

- EXPAND: 무엇을 추가하면 임팩트가 2배 되는가? 비용 증가는 정당한가? / SELECTIVE: 제외 기능 없이 경험이 완결되는가? 포함/제외 기준은 무엇인가?
- HOLD: 6개월 후에도 필요한가? 대기 비용은 무엇인가? / REDUCE: 무엇을 포기하는가? 줄여도 성공 지표를 측정할 수 있는가?

## 결론

근거 없는 결론 금지. 사용자 판단이 있어야 합니다.

- APPROVED: `$omc-plan`으로 설계합니다.
- REDUCE: 범위 축소 후 이 스킬을 다시 실행합니다.
- REJECT: 보류 사유를 `.omc/notepad.md`에 기록하도록 안내합니다.

```text
모드:
공통 5개:
선택 모드 질문 2개:
사용자 판단:
결론: APPROVED / REDUCE / REJECT
이유:
```

출력 후 자동 진입 금지. 자동으로 진행하지는 않습니다. 사용자가 다음 스킬을 명시할 때까지 멈춥니다.

모든 LLM 공통 출력 형식: 모드, 공통 5개, 선택 모드 질문 2개, 사용자 판단, 결론, 이유
입력 부족 시 중단: 근거 수치가 빠진 항목이 있으면 결론을 내리지 않고 누락 항목만 재요청
