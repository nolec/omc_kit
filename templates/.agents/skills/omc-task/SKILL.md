# OMC TDD 파이프라인

구현 작업을 7단계 체크포인트 양식으로 실행합니다. **건너뜀 금지.**

> **이 스킬을 쓰면 안 되는 상황**:
> - 계획(omc-plan)이 아직 완료되지 않은 경우 → `$omc-plan` 먼저
> - 버그 원인이 불명확한 경우 → `$omc-investigate` 먼저

---

## 전제 조건

> **AI는 아래 커맨드를 실행하고 결과를 확인한 후 파이프라인을 시작한다. 건너뛰지 않는다.**

```bash
python3 scripts/omc_guard.py require --for "task" --target . 2>/dev/null
```

---

수집 결과 연결:
- `omc_guard.py` 결과 → 세션 확정 여부 판단

> 각 항목에 답하기 어려우면 `N/A — [이유]` 형식으로 기재한다. 빈칸으로 두지 않는다.

## 이후 액션

| 상태 | 다음 단계 |
|---|---|
| PHASE 1 컨펌 안 됨 | 목표·DoD 재조율 후 재시작 |
| PHASE 3 FAIL 출력 없음 | 실제 FAIL 출력 첨부 후 RED 등록 재실행 |
| PHASE 4 GREEN 실패 | 구현 수정 후 PHASE 4 재실행 |
| PHASE 6 TDD GATE 실패 | 테스트 수정 후 PHASE 6 재실행 |
| PHASE 1~7 모두 완료 | → `$omc-review` 또는 `/ship` |

---

## 규칙
- 체크박스를 채우지 않으면 다음 단계로 진행하지 않습니다
- RED 등록(실제 FAIL 출력 첨부) 없이 구현 파일 생성 금지
