# 헤드리스 Gemini 실행 결과 판단 기준
날짜: 2026-06-12
태그: gemini headless tool-error exit-code investigation

## 증상
run_shell_command 툴 에러 로그가 출력되어 gemini-3-flash-preview가 미접근 상태로 오판

## 원인
exit code 0 + model stats totalErrors:0 임에도 tool fail 로그만 보고 모델 접근 실패로 오해. run_shell_command 차단은 headless 권한 제어로 정상 동작임

## 적용된 규칙
헤드리스 실행 결과는 exit code + stats.totalErrors를 함께 확인한다. run_shell_command fail이 있어도 totalErrors:0이면 모델이 정상 응답한 것이다. tool fail != model inaccessible

## 검증 커맨드
gemini -m <model> -p '1+1' --output-format json 실행 후 exit code와 JSON stats.models.<model>.api.totalErrors 확인
