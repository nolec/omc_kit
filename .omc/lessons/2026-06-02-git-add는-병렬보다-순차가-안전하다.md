# git add는 병렬보다 순차가 안전하다
날짜: 2026-06-02
태그: git,staging,lock,workflow

## 증상
동시에 여러 git add를 실행했을 때 .git/index.lock 충돌이 나면서 staged 정리가 끊겼다.

## 원인
같은 저장소 인덱스를 병렬로 갱신하면 lock 파일이 생겨 후속 add가 실패한다.

## 적용된 규칙
같은 저장소에서 스테이징은 병렬로 묶지 말고 순차로 실행한다. 특히 ignored 파일을 -f로 추가할 때는 단일 명령으로 처리한다.

## 검증 커맨드
git add 후 git status -sb 에서 index.lock 오류 없이 staged 상태가 유지되는지 확인
