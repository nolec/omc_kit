# 다음 스킬 추천은 파이프라인 기본값보다 현재 병목과 사용자 의도를 우선한다
날짜: 2026-06-16
태그: skills,recommendation,workflow,ux

## 증상
plan, review 같은 스킬 이후 다음 추천이 실제 문맥보다 plan→task, review→ship 같은 기본 파이프라인 순서로 쏠려 사용자가 추천 품질을 의심했다.

## 원인
스킬의 다음 추천 규칙이 현재 병목과 사용자 의도보다 기본 다음 단계 경로를 더 강하게 우선했고, 사용자 선택 대기를 정상 분기로 충분히 취급하지 않았다.

## 적용된 규칙
다음 스킬 추천은 파이프라인 기본값보다 현재 병목과 사용자 의도를 먼저 판정한다. 구현 승인, 범위 불확실성, 사용자 선택 대기 여부를 확인한 뒤에만 task, review, ship 같은 실행 스킬을 추천한다.

## 검증 커맨드
전략성 스킬에는 사용자 선택 대기 분기를 넣고 docs/verification_checklist.md의 다음 스킬 추천 품질 검증 시나리오로 PASS/FAIL을 확인한다.
