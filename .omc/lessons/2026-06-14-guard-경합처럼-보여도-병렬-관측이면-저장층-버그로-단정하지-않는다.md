# guard 경합처럼 보여도 병렬 관측이면 저장층 버그로 단정하지 않는다
날짜: 2026-06-14
태그: omc,guard,state,latest,parallel,workflow

## 증상
sync-session 직후 state status나 guard가 직전 analysis/code_review 세션을 읽는 것처럼 보여 저장층 또는 guard 버그로 의심했지만, 순차 재현 테스트에서는 latest.json과 state status가 일관되게 최신 세션을 가리켰다.

## 원인
병렬 도구 호출 또는 시차가 있는 출력 두 개를 같은 시점의 단일 진실로 해석해 저장층/guard 경합 버그로 과대 판정했다.

## 적용된 규칙
guard 또는 latest session 이상 징후를 보면 먼저 순차 시나리오로 sync-session -> latest.json -> state status를 재현하고, 순차 재현이 실패할 때만 저장층이나 guard를 수정한다. 병렬 조회 불일치는 버그 근거로 쓰지 않는다.
