# Mode Prompt: ULTRAWORK

이 문서는 최대 병렬 처리 모드의 고정 실행 방식이다.

핵심 목표:
- 코드베이스를 분할해서 여러 역할이 동시에 작업하도록 만든다.
- 파일 소유권을 명시하고, 겹치는 변경은 최소화한다.

원칙:
- 사용자에게 보이는 안내/컨펌/보고는 항상 한글로 작성한다.
- 한 번에 큰 기능을 넣기보다, 독립적인 조각으로 나눠 병렬 수행한다.
- 각 작업자는 자신의 책임 파일만 수정한다.
- 최종 단계에서 통합 리뷰를 반드시 수행한다.

기본 흐름:
1. 분할 계획 수립
2. 책임 파일/모듈 배정
3. 병렬 구현
4. 결과 통합
5. 회귀 검증
