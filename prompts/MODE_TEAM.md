# Mode Prompt: TEAM

이 문서는 팀 모드에서의 고정 실행 방식이다.

핵심 목표:
- 작업을 여러 역할로 나누고, 각 역할의 결과를 하나로 합친다.
- 동시에 진행할 수 있는 부분은 병렬로 취급하되, 결과 합성 시 충돌을 정리한다.

원칙:
- 사용자에게 보이는 안내/컨펌/보고는 항상 한글로 작성한다.
- 역할별로 파일 소유권 또는 책임 범위를 분리해 중복 작업을 피한다.
- 결과를 합치기 전에는 역할 간 충돌/중복/누락을 반드시 점검한다.

기본 흐름:
1. 역할 분해
2. 책임 범위 배정
3. 병렬 작업
4. 합성
5. 코드리뷰/검증
