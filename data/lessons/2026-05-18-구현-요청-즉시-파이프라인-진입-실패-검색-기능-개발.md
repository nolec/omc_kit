# 구현 요청 즉시 파이프라인 진입 실패 (검색 기능 개발)
날짜: 2026-05-18
태그: tdd,pipeline,hooks,next.js,sidebar

## 증상
CONTRACT 없이 구현 시작 → 테스트 없는 구현 → 사용자 사용 중 버그 발견 → 반응적 수정 반복 (탭 사라짐, 빈 결과 시 전체 표시, 검색어 리셋 등)

## 원인
1) 구현 요청 수신 시 CONTRACT 양식을 출력하지 않고 바로 분석/구현 진입
2) 복잡한 훅(router, window, IME)은 테스트 작성이 번거롭다는 암묵적 판단으로 RED 단계 회피
3) 파이프라인 가드가 차단했을 때 테스트 작성 대신 allow + Shell 명령으로 우회
4) 빠른 수정 압박으로 재현 테스트 없이 패치 반복

## 적용된 규칙
1) 구현 요청 즉시 CONTRACT 양식 출력 → 사용자 컨펌 대기 필수
2) 복잡한 훅일수록 renderHook + mockRouter로 테스트 먼저
3) 파이프라인 가드 차단 = 테스트 작성 신호, 우회 금지
4) 버그 수정 시 재현 테스트 RED → 수정 → GREEN 순서 필수
