# OMC Skill Optimization Stop Rules

지금 멈춰야 하는 신호
- 같은 혼선이 순차 재현 없이 관측 한 번으로만 보일 때
- `REVISE`나 `HOLD`가 스킬 결함이 아니라 문제 정의 불안정에서 나온다고 판단될 때
- 최근 스킬 변경 직후인데 추가 최적화 이유가 토큰 절감 감각 외에 뚜렷하지 않을 때
- 코드 변경보다 `.omc` 산출물만 늘고 있을 때

다시 시작해도 되는 신호
- 같은 문제가 순차 재현으로 2회 이상 반복 재현될 때
- 특정 스킬에서 `REVISE`가 같은 이유로 누적될 때
- 벤치마크나 실제 사용 로그에서 토큰, 속도, 정확도 손해가 분명할 때
- contract test로 줄이면 안 되는 핵심 마커를 먼저 잠글 수 있을 때

운영 규칙
- 반복 재현 없는 상태에서는 스킬 최적화보다 관측 프로토콜 점검을 우선한다
- 토큰 절감보다 판단 품질 손상 가능성을 먼저 본다
- 최적화 재개 전에는 다음 3가지를 먼저 적는다:
  1. 재현 시나리오
  2. 유지할 안전 마커
  3. 중단 조건
