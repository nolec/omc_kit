# OMC Benchmark

세계 1등 비교 분석 스킬입니다. 사용자에게 보여줄 것: BENCHMARK 설정 / 갭 분석 / 차별화 포인트 / 우선순위 TOP 1 / 다음 액션. 시스템이 암묵적으로 처리: 비교 대상은 사용자가 직접 확정, 임의 확정 금지, 근거 낮으면 판정 보류, 자동 구현 진입 금지

## Phase 0. 컨텍스트

```bash
python3 scripts/omc.py state status --target .
```

OMC 세션이 stale이어도 사용자의 현재 요청을 우선합니다. 부족한 정보는 `N/A — 이유`로 씁니다.

## Phase 1. BENCHMARK 설정

- 분석 기능:
- 비교 대상: 세계 1등 제품명 / 비교 관점: UX, 기능, 성능, 모바일, 운영, 가격 등.
- 우리 현재 수준 근거: 코드/문서/스크린샷/사용자 설명. 없으면 `N/A — 현재 제품 상태 근거 없음`.
- 비교 대상 미지정 시 후보 3개 이하를 선택지로 제안하고 사용자 선택 전 임의 확정 금지.
- 검증 상태: 미검증 / 직접 검증됨 / 공식 출처 확인됨.

비교 대상과 우리 현재 수준 근거가 없으면 갭 판정을 추정하지 않습니다.

## Phase 2. 갭 분석

최소 3개 항목을 같은 형식으로 씁니다.

```text
항목:
  1등 수준:
  우리 수준:
  갭 판정: 우위 / 동등 / 열위 / N/A
  갭 크기: 큼 / 중간 / 작음 / N/A
  근거 수준: 높음 / 중간 / 낮음
```

근거 수준이 낮음이면 반드시 `직접 검증 필요`를 붙입니다.

## Phase 3. 차별화 포인트

- 1등이 약한 것 / 우리가 유리한 맥락 / 차별화 포인트: 구체적 제안 1개

차별화 가설만 있고 근거가 약하면 제품 판단을 먼저 합니다.

## Phase 4. 우선순위 TOP 1

- 우선순위 TOP 1:
- 이유:
- 근거 수준:

갭 크기가 크고 근거 충분한 항목을 우선합니다. 근거가 낮으면 TOP 1 대신 검증 과제로 둡니다.

## Phase 5. 다음 액션

- 근거 수준 낮음 다수 → 실제 제품 검증
- 차별화 가설 검증 필요 → `$omc-office-hours`
- 근거 충분 + TOP 1 확정 → `$omc-plan`

분석 이후 자동 구현 진입 금지. 사용자가 다음 스킬을 명시할 때까지 멈춥니다.
