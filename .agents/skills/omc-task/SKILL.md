# OMC TDD 파이프라인

구현 작업을 7단계 체크포인트 양식으로 실행합니다. **건너뜀 금지.**

## 전제 조건

```bash
python3 scripts/omc_guard.py require --for "task" --target .
```

## 파이프라인 양식

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PHASE 1 ▸ CONTRACT
목표   : _______________
범위   : _______________
DoD    : _______________
☐ 사용자 컨펌 완료
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PHASE 3 ▸ RED  🔴
테스트 파일   : _______________
테스트 케이스 : _______________
☐ 실제 FAIL 출력 첨부 완료
  python3 scripts/omc_pipeline_guard.py red-done <테스트파일>
☐ RED 등록 완료
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PHASE 4 ▸ GREEN  🟢
구현 파일  : _______________
☐ 테스트 PASS 확인
☐ 기존 테스트 회귀 없음
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PHASE 6 ▸ TDD GATE
  python3 scripts/omc_tdd_check.py --staged
☐ 반환값 0 확인
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
PHASE 7 ▸ COMPOUND ENGINEERING
  python3 scripts/omc_lesson.py add -i
☐ 교훈 없음 / ☐ 교훈 기록 완료
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 규칙
- 체크박스를 채우지 않으면 다음 단계로 진행하지 않습니다
- RED 등록 없이 구현 파일 생성 금지

## 리뷰 지적 반영 전용 템플릿 (선택)
`omc-review`에서 나온 이슈를 바로 반영할 때 아래 CONTRACT를 먼저 채웁니다.

```
[REVIEW-FIX CONTRACT]
리뷰 출처     : _______________ (예: omc-review YYYY-MM-DD)
수정 대상 이슈 : _______________ (치명/중대/경미)
변경 범위     : _______________ (파일/함수)
비목표        : _______________ (이번 턴에서 제외)
검증 기준     : _______________ (재현 테스트 + 회귀 테스트)
```
