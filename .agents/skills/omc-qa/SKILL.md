# OMC QA

수동 QA 체크리스트 전용입니다. 구현 후 사용자 검증을 돕는 목적이며 코드 리뷰를 하지 않습니다. 배포 게이트를 대신하지 않습니다.

```bash
git diff --stat HEAD 2>/dev/null | head -20
python3 scripts/omc.py state status --target .
```

- diff 또는 plan이 없고 입력 부족이면 `고정 QA 생성 불가`로 멈춥니다.
- 사용자에게 보여줄 것: 정상 흐름 / 예외 흐름 / 회귀 포인트 / 우선순위 높은 QA 5개 / 다음 추천
- 시스템이 암묵적으로 처리: 변경 범위 읽기 / 화면·역할 추론 / 중복 항목 제거

- 변경 기능:
- 영향 화면/흐름:
- 사용자 유형:
- 확인 환경:

## 출력

- 정상 흐름
- 예외 흐름
- 회귀 포인트
- 우선순위 높은 QA 5개
- 실행 결과 아님

## 경계

- `omc-plan`의 구현 전 테스트 설계를 가져오지 않습니다.
- `omc-review`의 코드 리스크 검토를 가져오지 않습니다.
- `omc-ship`의 배포 승인/게이트를 가져오지 않습니다.

## 다음 추천

- 주추천 1개: 구현 후 검수면 `$omc-review`, 배포 직전 최종 점검이면 `$omc-ship`
- 구현 전이거나 입력 부족이면 `$omc-plan`
- 자동으로 진행하지는 않습니다.
