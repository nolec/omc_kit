# OMC 제품 사고 먼저 (코딩 전)

6개 강제 질문에 답하지 않으면 구현 단계로 넘어갈 수 없습니다.

```
Q1. 이 기능을 쓸 사람이 정확히 누구인가?
    ("모든 사용자"는 답이 아닙니다)
    → _______________

Q2. 지금 그 사람이 겪는 실제 고통은 무엇인가?
    → _______________

Q3. 이 기능이 완성됐을 때 성공 기준은?
    (측정 가능한 지표로)
    → _______________

Q4. MVP를 한 문장으로
    → _______________

Q5. 이 기능에서 명시적으로 제외하는 것은?
    → _______________

Q6. 10점짜리 버전은?
    → _______________

☐ 6개 질문 답변 완료
```

## 이후 → `$omc-plan`으로 이어가기

## 다음 스킬 추천 (필수 1개)
답변 완료 후 아래 중 **정확히 1개**를 추천해 출력합니다.
- 요구사항이 아직 모호함 → `$omc-brainstorm`
- 구현 범위/DoD 확정 필요 → `$omc-plan`
- 이미 실패/오류가 있음 → `$omc-investigate`
